The DockerPedia Ontology: Overview back to ToC

This ontology has the following classes and properties.

Classes

  • A file containing the instructions required to build an image (docker:SoftwareImage) in the Docker framework
  • A layer of a Docker image are a set files generated from running a command from a DockerFile.
  • a mistake in a software component that leaves it open to exploitation by an adversary
  • A modification of a layer in a DockerFile
  • A name that identifies either a unique object or a unique class of objects.
  • A read-only template that contains a set of instructions for creating a container.
  • A serialized copy of the state of a software stored in some non-volatile form such as a file.
  • A software operating system repersentation
  • An identifier of a software image
  • security revision of a software package
  • Set of individual software files or resources that are packed together as a software collection that provides certain functionality as part of a larger system.
  • The way in which files are named and where they are placed logically for storage and retrieval.
  • version of a software image
  • Version of a software package

Object Properties

  • A docker file has software images
  • A DockerFile builds a SoftwareImage since the DockerFile is a description of that image
  • a package version is a versuon of a software component
  • a package version modifies a layer
  • a security revision fixes a package version
  • a security revision fixes a vulnerability
  • a software image contains several packages
  • a software layer use an Operating System
  • a vulnerability that is present in an operating system
  • An identifier of a software image
  • an image layer modifies a package version
  • An OS has a file system in which install software
  • one OS may have installed several software packages on it
  • one package version may have a security revision fixing some vulnerability
  • One package version may have one or more software vulnerabilities
  • One Software image may be composed of several layers
  • one software images may be composed of several layers
  • one software layer is part of a software image
  • Operating Systems are affected by software vulnerabilities
  • software components are package on an Operating System
  • software images deploy on FileSystems

Data Properties

  • An identifier of a software image
